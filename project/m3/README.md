# Milestone 3 Repository Index

## Simulator and Co-Simulation Reproduction
* **Simulator Used:** Icarus Verilog (v11.0+)
* **Reproduction Command:** Run `make sim` from the repository root directory. This executes the following underlying command: `iverilog -g2012 -s tb_top -o project/m3/sim/cosim.vvp project/m3/rtl/compute_core.sv project/m3/rtl/interface.sv project/m3/rtl/top.sv project/m3/tb/tb_top.sv && vvp project/m3/sim/cosim.vvp`
* **Dependencies:** Icarus Verilog (`iverilog`) and `make`. No special preprocessing required.

## Synthesis Tool and Reproduction
* **Synthesis Tool:** OpenLane 2 (Release Tag: `2.0.0-b3`)
* **Configuration File:** `project/m3/synth/config.json`
* **Reproduction Command:** Run via the standard OpenLane 2 docker container: `openlane --dockerized project/m3/synth/config.json`
* **Environment Variables:** Default Docker environment; no custom variables required.

## Directory Contents Catalog

**Root / Docs**
* `project/m3/README.md` - This index file cataloging the M3 deliverables and reproduction steps.
* `project/m3/synthesis_notes.md` - Narrative detailing the synthesis results, iterative failures, and M4 plans.

**RTL (`project/m3/rtl/`)**
* `project/m3/rtl/top.sv` - The integrated top module instantiating the interface and compute core.
* `project/m3/rtl/compute_core.sv` - The MAC compute core hardware design.
* `project/m3/rtl/interface.sv` - The AXI4-Stream interface logic.

**Testbench (`project/m3/tb/`)**
* `project/m3/tb/tb_top.sv` - The end-to-end co-simulation testbench driving the integrated module using the host protocol.

**Simulation Data (`project/m3/sim/`)**
* `project/m3/sim/cosim_run.log` - The captured simulation transcript showing the final "PASS" assertion.
* `project/m3/sim/cosim_waveform.png` - Annotated waveform showing write, compute, and read transactions.
* `project/m3/sim/cosim.vcd` - The raw VCD waveform data file generated by the simulation.
* `project/m3/sim/cosim.vvp` - The compiled simulation executable generated by Icarus Verilog.

**Synthesis Data (`project/m3/synth/`)**
* `project/m3/synth/config.json` - The OpenLane 2 configuration file dictating the 10.0ns clock target and area specs.
* `project/m3/synth/openlane_run.log` - Full captured stdout/stderr log of the successful OpenLane run.
* `project/m3/synth/timing_report.txt` - STA report detailing the -3.49 ns WNS and setup violations.
* `project/m3/synth/area_report.txt` - Total cell area breakdown (11,902.7 um^2, 4165 cells).
* `project/m3/synth/critical_path.md` - Start point, end point, and logic description of the worst timing path.
* `project/m3/synth/power_report.txt` - Documentation of the failure to generate the SAIF power profile.
